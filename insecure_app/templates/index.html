This mini Flask application is purpose-built to demonstrate two classic SQL injection vulnerabilities. Explore the pages below to see how unsanitised input can break security.

  • Login Demo: Shows how an attacker can bypass authentication using the OR trick.
  • Search Demo: Highlights a UNION SELECT attack that reveals data from another table.

Each page displays the raw SQL generated for your request so you can trace exactly how the attack works. In real systems you must always use parameterised queries or an ORM to avoid these issues.
